No. of rows in each table

Deprecated but so far it works on all versions. This makes use of the fact that the number of rows are stored with each index. o.xtype = 'U' selects only user tables and i.indid < 2 makes sure we select only one index per table. This can be a clustered index (1) or a heap index(0).

SELECT o.name, i.rowcnt FROM sysindexes AS i INNER JOIN sysobjects AS o ON i.id = o.id WHERE i.indid < 2 AND o.xtype = 'U';

Indexes

Heap
Records are not stored in any particular order
Clustered
Records are stored according to this index (often primary key)
Non-Clustered
Only links to the records. Does not affect the way the records are stored
